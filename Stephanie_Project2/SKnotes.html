Basics of Web and HTML

The World Wibe Web is a collection of documents, most are made up of HTML. The browser reads, interprets, and displays these documents. Servers are computers that host files that make up the web. Computers make requests via the internet to servers, the internet and servers communicate through http. HTTP is the main protocol of the web.

Coding is when programmers write text in a language computers understand, so the computer can follow the instructions. Computers are stupid, even a small mistake like a typo can break the whole program.

Basic HTML vocabulary:
Tag - HTML tags are always contained in angle brackets. Most tags have opening and closing tags (designated with a forward-slash), but some tags called void tags do not require a closing tag. There are inline tags and block tags - block tags create an invisible box or container around the content that has a height and width, while inline tags do something within the line.
Element - An HTML element refers to everything within a set of opening and closing tags.
Attribute - This is a property of an HTML element, such as the source in an image tag.

Creating a Structured Document with HTML

HTML (HyperText Markup Language) is the structure of a webpage. HTML is the standard markup language used to create webpages.

DOM (Document Object Model) refers to the tree-like structure of a webpage. DOM is a cross-platform and language-independent convention for representing and interacting with objects in markup languages. The nodes of every document are organized in a tree structure, called the DOM tree.

Websites are all made up of boxes, with boxes inside those boxes, and more boxes inside those. Box-ception. When reading the HTML you can see the "boxes" by the indentation in the code - the further indented the code is the deeper the visual box is nested.

Adding CSS Style to HTML Structure

CSS (Cascading Style Sheets) creates the style of a webpage. CSS is a style sheet language used for describing the look and formatting of a document written in a markup language.

The Cascading part of CSS means that rules are applied to the elements they directly match as well as that element's child elements, like a waterfall will wet every stone from the top to the bottom. This idea is known as inheritance. If a child element has multiple, overlapping rules defined for it, the more specific rules take precedence.

Avoiding repetition is very important in programming. It helps avoid errors when you don't have to rewrite the same thing over and over again - there is always the risk of typos! Avoiding repetition also helps with creating consistency. With CSS, you can apply the same style to everything of the same element, like I have done with all of the div tags with the class "lesson" here. Because of that, it is now very easy for me to make changes to all of the lessons' style at once.

Telling Computers What to Do

A computer is a universal machine, that means it can be programmed to do anything we want it to do, unlike machines like a vacuum cleaner that can only do their one function. A program is a precise sequence of steps for a computer to follow. Some examples of programs are web browsers, video games, and instant messaging.

We create new languages for computers rather than using natural languages like English because of the ambiguity and verbosity of natural languages. We want to be able to tell the computer precicely what to do without requiring a large amount of text. Programming languages have grammar rules just like natural languages, but we must follow the rules exactly for the program to work. An expression in code is a legal statement, one that follows the grammar rules of the programming language.

Python is a programming language. When you run Python code, a Python interpreter converts the code into instructions for the computer to execute.

Variables and Strings

A variable is a way to give a name to a value so you can easily reference and use that value later. You assign a value to a variable using the equal sign, such as my_age = 23. Variables are useful in storing data and also in referencing that data in a way that makes sense to humans reading the code. Using variables also allows us to easily change the value of something, like days +=1.

In Python, 5 is a number (or integer), while "5" is a string. Everything within quotes (either single quotes or double quotes) is a string. Running the code 2 + 3 would give us 5, but the code "2" + "3" gives "23" because it is concatenating (squishing together) the two strings.

Functions

Functions take input, do something with the input, and produce output. A function named "add" might take two numbers as input and output the sum of them. Once you create a function, you can reuse it in the code without having to redefine it each time, which helps to avoid repetition.

Functions are created by starting a line with def and following with the function name and the parameters/arguments in parenthesis, followed by a colon. The next lines will be the body of the function, and they will be indented. 

def add(a, b):
      return a + b

The return keyword tells Python what the function needs to output. A function without a return statement doesn't do anything that the user can see.

To use or call a funtion you type the name of the function with the arguments/parameters you want to use in parenthesis, like add(2, 3)

Repetition, Loops, and Structured Data

If statements let us control what part of the code is executed. If the condition is True, the indented block of code will execute. If not the interpreter moves on without executed the block under the if statement. An else statement under an if statement is executed when the if statement was not True.

While statements will loop through the indented block of code until the condition is no longer met. Both while and if statements use comparision operators, like greater than or less than, to return the Boolean values True and False.

Lists are sequences of data contained in square brackets [ ] and can contain any combination of numbers, strings, or Boolean values. Lists can even contain other lists. For loops allow you to iterate over lists. 

for dog in doggy_daycare:

How to Solve Problems

The first step to solving any problem is to understand the inputs and outputs. The inputs are anything the program will be responsible for processing, while the outputs are the desired response we want the function to provide.

Before you start coding, it is often useful to work through some examples by hand and develop a simple mechanical solution that we can base the function off of. Then once you start coding, develop the code in small parts and test as you go. It's easier to fix problems as they arise than to have to find and fix all the problems at the end!
